# Crypto Spreads Dashboard

Дашборд для моніторингу basis spread (perpetual vs spot) на всіх USDT-парах
Binance в реальному часі. Без бекенду — все працює прямо в браузері через
публічне API Binance.

## Як запустити

**Варіант 1 — просто відкрити файл**:

Відкрий `crypto_dashboard/index.html` подвійним кліком у браузері. Працює
з будь-якого сучасного Chrome / Edge / Firefox / Safari.

**Варіант 2 — через локальний сервер** (рекомендую, інколи браузер
блокує fetch з file://):

```bash
cd crypto_dashboard
python3 -m http.server 8001
# відкрий http://localhost:8001
```

## Що показує

| Колонка | Що це |
|---|---|
| **Символ** | Тикер (BTC, ETH, SOL...) |
| **Спот** | Поточна ціна на спот-ринку Binance |
| **Perp** | Поточна ціна perpetual ф'ючерса |
| **Basis %** | (Perp / Spot − 1) × 100 — головна метрика |
| **Funding %** | Поточна funding rate (платиться кожні 8 годин) |
| **Funding річн.** | Funding × 3 × 365 — приблизна річна дохідність якщо стояти в шорт perp + лонг спот |
| **Обсяг 24h** | Об'єм торгів за добу (USD) — фільтр ліквідності |
| **Зміна 24h** | Зміна ціни за добу (%) |
| **Сигнал** | 🟢 з'являється коли \|basis\| ≥ поріг |

## Налаштування

- **Пошук** — фільтр по тикеру (наприклад "BTC" покаже тільки BTC-пари)
- **Поріг basis** — від якого \|basis\| показувати сигнал (за замовчуванням 0.3%)
- **Мін. обсяг** — відсікає неліквідні пари (за замовчуванням $1M / 24h)
- **Сортування** — за |basis|, basis, funding, обсягом або алфавітом
- **Інтервал** — як часто оновлювати (5 / 10 / 30 / 60 секунд)
- **Звуковий сигнал** — короткий beep коли з'являється новий сигнал
- **Pause** — призупинити автооновлення

Всі налаштування зберігаються в localStorage браузера.

## Кольорове кодування рядків

- **Сірий** — норма, спред нижче порогу
- **Жовтий** — увага, спред наближається до порогу (60–100%)
- **Зелений** — сигнал, |basis| ≥ поріг
- **Червоний** — екстремум, |basis| ≥ 2× поріг (рідкісна подія)

## Що робити коли побачив сигнал

Базовий сценарій для basis > +0.3% (perp дорожчий за спот):

1. **Перевір funding rate** — має бути позитивний
2. **Перевір обсяг** — не торгуй пари з обсягом < $1M (slippage з'їсть прибуток)
3. **Перевір новини** — basis може стрибнути через подію (CPI, FOMC, hack)
4. **Відкрий дві позиції одночасно**:
   - Spot: КУПИТИ X токенів
   - Perp: ШОРТ X токенів (плече 1x, не більше)
5. **Чекай конвергенції** — basis повертається до 0 зазвичай за 1-7 днів
6. **Закрий обидві ноги** коли |basis| < 0.1%

Детальніше — див. історію розмов або зроби бектест в `gold_silver_arb/`.

## Обмеження

- Тільки Binance USDT perpetuals (найбільш ліквідні)
- Тільки публічні API (без авторизації, тому без власних угод)
- Funding rate — це **останній** snapshot, не точна прогноза наступного
- Дані оновлюються раз на 5-60 секунд (за вибором користувача)

## Що додати наступним кроком

Якщо буде корисно — можна розширити:

- Binance + Bybit + OKX порівняння (cross-exchange spread)
- Telegram-бот для push-сповіщень на телефон
- Історія сигналів (журнал, статистика "якби входив завжди при сигналі")
- Stablecoin depeg окремою вкладкою
- Кореляційна матриця крипто-пар для pairs trading
